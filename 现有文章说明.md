## 💡 什么时候需要迁移旧文章？

### 需要迁移的情况：
1. **想要编辑某篇旧文章**
   - 需要修改内容
   - 想用Markdown格式编辑更方便

2. **图片路径有问题**
   - 图片显示不正常
   - 需要统一图片管理

3. **想统一管理**
   - 所有文章都想在Obsidian中管理
   - 方便统一编辑和维护

### 不需要迁移的情况：
1. ✅ 文章内容不需要修改
2. ✅ 图片显示正常
3. ✅ 只想发布新文章

## 🔄 如果将来需要迁移

### 迁移步骤（仅供参考）：

1. **提取HTML内容**
   - 打开HTML文件
   - 复制正文内容

2. **创建Markdown文件**
   - 在Obsidian中创建新文件
   - 添加Front Matter：
     ```markdown
     ---
     title: "文章标题"
     date: YYYY-MM-DD
     description: "文章描述"
     ---
     ```
   - 粘贴内容，转换为Markdown格式

3. **处理图片**
   - 找到HTML中的图片路径
   - 复制图片到Markdown文件同目录或attachments文件夹
   - 更新图片引用

4. **重新生成**
   ```bash
   ./scripts/deploy_blog.sh "/path/to/new/article.md"
   ```

5. **替换旧文件**
   - 备份旧HTML文件
   - 用新生成的HTML替换
