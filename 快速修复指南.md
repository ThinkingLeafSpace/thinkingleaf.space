# 快速修复指南

## 🎯 博客文件名问题

如果你的博客文件名变成了 `2022-08-21-2022-08-21.html` 这种"日期-日期"的格式，使用以下方法修复：

---

## ⚡ 快速修复（推荐）

```bash
cd "/Users/qianny/Nutstore Files/ThinkingLeafSpace 20250704com/ThinkingLeafSpace 20250924com"
python3 scripts/fix_blog_slugs.py
```

**这个脚本会自动：**
1. ✅ 检测所有"日期-日期"格式的文件
2. ✅ 从HTML中提取标题
3. ✅ 使用 `title_slug_mapping.json` 转换为英文slug
4. ✅ 重命名文件
5. ✅ 更新所有引用链接

---

## 📝 添加新标题映射

当你有一篇新文章需要英文slug时：

### 1. 编辑映射文件

打开 `scripts/title_slug_mapping.json`，添加一行：

```json
{
  "你的中文标题": "your-english-slug",
  "另一篇文章标题": "another-english-slug"
}
```

### 2. 运行修复脚本

```bash
python3 scripts/fix_blog_slugs.py
```

## 🆘 常见问题

**Q: 脚本提示"没有需要修复的文件"？**  
A: 说明所有文件都正常，没有问题！

**Q: 如何查看映射表？**  
A: 打开 `scripts/title_slug_mapping.json`

**Q: 自动生成的slug不好，可以改吗？**  
A: 可以！编辑 `title_slug_mapping.json` 添加你想要的slug，然后重新运行脚本。

**Q: 修复后网站会坏掉吗？**  
A: 不会！脚本会自动更新所有引用链接，包括 index.html、blogs.html、sitemap.xml 等。
