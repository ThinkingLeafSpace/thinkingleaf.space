# 如何获取Obsidian文件的完整路径

## 方法1: 拖放文件（最简单 ⭐推荐）

1. 打开终端
2. 输入：`./scripts/deploy_blog.sh `（注意末尾有个空格）
3. 在Finder中找到你的Markdown文件
4. 将文件拖到终端窗口
5. 终端会自动显示完整路径
6. 按回车执行

**示例：**
```
bash-3.2$ ./scripts/deploy_blog.sh [拖放文件]
bash-3.2$ ./scripts/deploy_blog.sh /Users/qianny/Documents/Obsidian/文章.md
```

## 方法2: 在Obsidian中获取

### macOS方法：
1. 在Obsidian中右键点击你的Markdown文件
2. 选择"在Finder中显示"或"在系统文件管理器中显示"
3. 在Finder中，右键文件 → "显示简介" 或按 `Cmd+I`
4. 在"位置"中可以看到完整路径

### 使用Obsidian命令：
1. 打开命令面板 (`Cmd+P`)
2. 搜索 "Copy file path" 或"复制文件路径"
3. 复制路径后，在终端中使用

## 方法3: 使用终端查找

如果你知道文件名，可以在终端中搜索：

```bash
# 在当前目录下搜索
find . -name "文件名.md" -type f

# 在整个用户目录搜索
find ~ -name "文件名.md" -type f 2>/dev/null

# 在特定目录搜索（例如Documents）
find ~/Documents -name "*.md" -type f
```

## 方法4: 使用Tab键自动补全

在终端中输入部分路径，然后按Tab键自动补全：

```bash
./scripts/deploy_blog.sh ~/Documents/Obsid[按Tab]
# 会显示可能的补全选项
```

## 常见路径示例

Obsidian文件通常在这些位置：

```bash
# 默认位置
~/Documents/Obsidian/
~/Library/Mobile Documents/iCloud~md~obsidian/Documents/

# 自定义位置
~/坚果云/Obsidian/
~/Dropbox/Obsidian/

# 完整示例
./scripts/deploy_blog.sh ~/Documents/Obsidian/我的博客/新文章.md
```

## 路径包含空格时

如果路径或文件名包含空格，需要用引号包裹：

```bash
./scripts/deploy_blog.sh "~/Documents/Obsidian/我的 博客/新 文章.md"
```

或者使用转义字符：

```bash
./scripts/deploy_blog.sh ~/Documents/Obsidian/我的\ 博客/新\ 文章.md
```

## 验证路径是否正确

在运行部署脚本前，可以先验证文件是否存在：

```bash
ls "/你的/文件路径.md"
```

如果文件存在，会显示文件信息；如果不存在，会显示错误。
