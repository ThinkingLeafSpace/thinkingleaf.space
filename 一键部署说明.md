## 📝 完整工作流程

### 🚀 最简单方式：在Obsidian中配置一键命令

> 详见：[OBSIDIAN快速部署配置.md](OBSIDIAN快速部署配置.md)

---

### 1️⃣ 在Obsidian中写文章

- 打开 `/Users/qianny/Nutstore Files/Qianny-obsidian/个人网站` 文件夹
- 使用模板创建：`Cmd+P` → "插入模板"
- 填写 title、date、category
- 保存（`Cmd+S`）

---

### 2️⃣ 一键部署（选择其一）

#### 【方式A】Obsidian中：

`Cmd+P` → "部署" → 回车

#### 【方式B】终端中：

1. 打开终端（`Cmd+空格`，搜索"终端"）
2. 复制下面的命令（一行一行运行）：

```bash
cd "/Users/qianny/Nutstore Files/ThinkingLeafSpace 20250704com/ThinkingLeafSpace 20250924com"

./scripts/deploy_blog.sh "文件路径.md"
```

**提示：** 拖拽文件到终端自动获取路径

---

### 3️⃣ 完成！

- ✅ 自动验证格式
- ✅ 自动转换HTML
- ✅ 自动更新列表
- ✅ 自动打开浏览器（可选）

## 💡 快捷方式（可选设置）

如果你想更简单，可以设置终端别名：

### 1. 打开终端配置文件

```bash
nano ~/.zshrc
```

### 2. 添加以下别名（复制粘贴）

```bash
alias blog-deploy='cd "/Users/qianny/Nutstore Files/ThinkingLeafSpace 20250704com/ThinkingLeafSpace 20250924com" && ./scripts/deploy_blog.sh'
```

### 3. 保存并重新加载

```bash
source ~/.zshrc
```

### 4. 以后只需要运行

```bash
blog-deploy "/path/to/article.md"
```

## ❓ 常见问题

### Q: 每次都要手动运行脚本吗？

**A:** 是的，需要在终端运行一次部署脚本

### Q: 能在Obsidian里一键部署吗？

**A:** 可以用Obsidian的插件（如Commander）配置命令

### Q: 脚本运行失败怎么办？

**A:** 看终端错误信息，通常是因为格式问题

### Q: 可以批量部署多篇文章吗？

**A:** 目前支持单篇，批量功能可后续添加
